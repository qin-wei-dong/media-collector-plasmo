# M7 发布收口设计

## 背景

M1-M6 已完成从采集、全屏素材库、收藏夹、分文件夹导出到大素材量效率增强的完整周期。当前 `main` 已发布到 v2.1.0 基线，点击扩展图标会直接打开全屏素材库。

发布前还需要完成一次产品定位、功能承诺、文档、商店素材和商业化口径收口。尤其是当前代码里仍保留抖音相关入口，但抖音采集暂时不作为发布承诺；收费体系也不应在首个公开版本里引入复杂账号、支付和授权系统。

## 目标

M7 的目标是把现有能力包装成一个可公开发布、可审核、可解释、可验证的 Chrome 扩展版本。

发布版定位为：

> 小红书素材采集与本地管理工具。

M7 交付后，用户和审核方应能清楚理解：

- 当前主要支持小红书笔记浮层 / 详情页素材采集。
- 素材和管理数据以浏览器本地存储为主。
- 抖音等更多平台属于后续探索，不在当前发布承诺中。
- 当前版本免费开放，后续是否做 Pro 取决于真实用户反馈。

## 非目标

- 不在 M7 实现抖音正式支持。
- 不新增账号体系。
- 不接入 Stripe、微信支付、支付宝或其他付费系统。
- 不做云同步、团队协作、云备份。
- 不做 Chrome Web Store 之外的安装分发流程。
- 不为了上架重写采集架构。

## 产品承诺

### 当前公开承诺

- 小红书笔记素材采集：点开笔记后，一键采集图片或视频素材。
- 本地素材库：全屏工作台、搜索、筛选、预览、批量选择。
- 收藏夹管理：新建、重命名、改色、置顶、移动素材。
- 导出：分文件夹批量导出、导出历史、失败项重试。
- 大素材量体验：500-1000 条素材下渐进渲染和基础性能优化。

### 暂不公开承诺

- 抖音采集。
- 小红书信息流 hover 采集。
- 云同步。
- 自动发布 / 搬运 / 去水印。
- 保证所有平台视频源永久可下载。

### 抖音能力处理

推荐 M7 采用“代码保留、发布承诺移除”的方式：

- README、CHANGELOG 新版本段、Chrome Web Store 文案不再写“支持抖音采集”。
- `package.json` description 改为小红书定位。
- `chrome.commands.collect_media.description` 不再写“悬停素材”，改成与当前小红书一键采集更一致的描述。
- 是否保留 `https://www.douyin.com/*` host permission 作为实施时的独立决策：
  - 更推荐发布版先移除该权限，降低审核解释成本和用户疑虑。
  - 如果保留权限，商店隐私说明必须解释它属于实验性平台适配，不作为当前主要能力。

## 收费体系

### M7 发布策略

M7 使用免费 Beta / 早期版策略：

- 不做登录。
- 不做付费墙。
- 不限制基础采集、收藏夹、导出历史和批量导出。
- 在 README 或后续官网中可以写“后续可能推出 Pro 功能”，但不在扩展内制造不可用入口。

这样做的理由：

- Chrome Web Store 不提供内置付费扩展 / 内购支付闭环，贸然收费意味着需要自建账号、支付、授权和客服系统。
- 当前最需要验证的是真实用户是否会高频使用“小红书素材采集 + 本地管理”。
- 免费发布可以更快获得安装、评价和使用反馈，为后续 Pro 定价提供依据。

### 后续 Pro 方向

若用户反馈明确，可以在 v2.2 / v3.0 评估 Pro：

- Free：
  - 小红书基础采集。
  - 本地素材库。
  - 基础收藏夹。
  - 基础批量导出。
- Pro：
  - 智能分类建议。
  - 重复素材检测。
  - 高级导出命名模板。
  - 更多平台支持。
  - 云同步 / 多设备备份。
  - 批量元数据整理。

初始可测试价格：

- 月付：19-29 元。
- 年付：129-199 元。

价格必须等用户反馈和使用数据确认后再落地，M7 不实现。

## 发布收口范围

### 文档一致性

需要修正所有对外或半对外文档中的旧承诺：

- `README.md`
  - 当前版本描述改为“小红书素材采集与管理”。
  - 使用方法移除抖音步骤。
  - 项目结构移除已删除的 `popup.tsx` / `popup.html` 描述。
  - 说明点击扩展图标直接打开全屏素材库。
  - 发布流程中保留 Chrome Web Store Action 说明。
- `CHANGELOG.md`
  - 新增 M7 发布收口段。
  - 标注对外定位从“小红书、抖音”收敛为“小红书优先”。
- `AGENTS.md`
  - 更新当前架构：不再把 `popup.tsx` 作为当前入口描述。
  - 抖音标注为暂不发布承诺。
- `package.json`
  - `description` 改为小红书定位。
  - manifest command 文案同步。

### Chrome Web Store 文案

需要准备一份商店素材草案，至少包括：

- 标题：素材采集助手 - 小红书素材管理
- 短描述：一键采集小红书笔记图片/视频，统一管理、收藏和批量导出。
- 长描述：
  - 说明点开小红书笔记后采集。
  - 说明素材保存在浏览器本地。
  - 说明支持收藏夹、预览、批量导出、导出历史。
  - 不承诺抖音。
- 权限说明：
  - `storage`：保存本地素材索引和收藏夹。
  - `downloads`：导出素材到本地下载目录。
  - `scripting` / `activeTab` / `tabs`：在小红书页面注入采集按钮和打开素材库。
  - `notifications`：展示采集和导出结果。
  - `contextMenus`：右键采集入口。

### 隐私说明

M7 需要准备隐私说明草案：

- 不上传用户素材列表到服务器。
- 不要求注册或登录。
- 素材索引、收藏夹、导出历史保存在 `chrome.storage.local`。
- 下载动作由用户主动触发。
- 不收集支付信息。
- 不对外分享用户浏览或采集数据。

### 验收清单

发布前必须手动验证：

- 安装 `build/chrome-mv3-prod`。
- 点击扩展图标能打开全屏素材库。
- 小红书图集采集成功。
- 小红书视频采集成功。
- 搜索、平台 / 类型筛选、收藏夹筛选可用。
- 预览图片 / 视频可用。
- 批量选择、取消全选、删除撤销可用。
- 收藏夹新建、重命名、改色、置顶、移动素材可用。
- 批量导出、导出历史、失败重试可用。
- 500 条样本数据下库页不白屏，滚动继续加载可用。
- 构建命令通过：`pnpm build`、`pnpm package`、`pnpm audit:a11y`。

## 风险与处理

### 风险 1：代码里保留抖音，文案不承诺

处理：发布文案只写当前稳定能力。实施时优先移除 Douyin host permission；若保留，需要在内部文档说明原因，并避免商店页展示抖音截图或承诺。

### 风险 2：免费版后续转 Pro 引发用户反感

处理：M7 不限制现有核心能力。未来 Pro 只增加高级能力，不把已经免费稳定提供的基础采集和本地管理突然锁住。

### 风险 3：商店审核关注权限过宽

处理：M7 优先做权限最小化。若发布版只承诺小红书，应移除 Douyin host permission，并在权限说明中逐项解释。

### 风险 4：README 与实际入口不一致

处理：M7 第一项任务就是修正文档结构，删除 `popup.tsx` 当前入口描述，改为 `background/index.ts` 的 action click 打开 `tabs/library.html`。

## 成功标准

- 所有对外文案不再承诺抖音采集。
- README / CHANGELOG / AGENTS / package manifest 与当前架构一致。
- 商店文案和隐私说明可以直接复制到 Chrome Web Store 后台。
- 权限列表与发布承诺匹配。
- 发布前验收清单完整，且构建、打包、a11y 审计通过。
- 不引入付费系统，不影响现有核心功能。
